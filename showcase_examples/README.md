# thought_bubble MCP Showcase Examples

This directory contains 13 comprehensive examples demonstrating all capabilities of the thought_bubble MCP tool.

## Quick Start

Open any HTML file in your browser. Each file is self-contained with:
- Interactive theme switcher (top-right corner)
- Responsive layout (works on mobile, tablet, desktop)
- Hover tooltips on charts
- Smooth animations and transitions

## Features Demonstrated Across All Examples

### Section Roles
- **metric**: Large KPI cards (dashboard, briefing, scorecard, dossier)
- **pull-quote**: Display-font blockquotes (magazine, editorial)
- **lead**: Wider opening paragraphs (magazine, briefing, dossier)
- **statement**: Large centred text (presentation, dossier, dialogue, editorial)
- **full-width**: Breakout visuals (magazine, scorecard, dialogue, editorial)
- **supporting**: Appendices, recommendations (comparison, briefing, scorecard, report, dialogue)

### Chart Types
- **Bar**: Exact comparison, categorical data
- **Line**: Trends over time, time series
- **Area**: Cumulative volume, growth trajectories
- **Donut**: Composition (max 6 categories)
- **Radial**: Multi-axis comparison, capability scores
- **Gantt**: Project timelines, migration roadmaps
- **Quadrant**: 2D positioning, risk matrices
- **Sankey**: Flow volumes, revenue streams
- **Treemap**: Hierarchical proportions, taxonomy

### Diagram Types (Mermaid)
- **Flowchart**: Processes, workflows, architecture
- **Sequence**: API interactions, authentication flows
- **ER Diagram**: Database schemas, data models
- **State**: State machines, transitions (not shown but supported)
- **Class**: Object relationships, data structures (not shown but supported)

### Chart Styling Options
- **emphasis**: glow (luminous halo), shadow (depth), lift (scale), none
- **curve**: smooth (time series), sharp (raw data), step (discrete), natural (organic)
- **animation**: stagger, draw, grow, fade, none
- **colorStrategy**: categorical, sequential, diverging, monochrome
- **annotations**: Data point labels with positioning (used extensively)

### Density Presets
- **compact**: Tight, information-dense (dashboard, briefing)
- **comfortable**: Balanced default (most examples)
- **spacious**: Generous whitespace (magazine, tutorial, editorial, minimal)

### Themes Used
1. **tokyo_night**: Modern tech, confident (comparison)
2. **dracula**: Dark, gothic, dramatic (dossier)
3. **gruvbox**: Warm, retro-modern (tutorial)
4. **solarized_dark**: Scientific precision (scorecard)
5. **solarized_light**: Readability, research (editorial)
6. **github_light**: Clean corporate (dashboard, sidebar)
7. **github_dark**: Modern developer (briefing)
8. **professional**: Polished business (report)
9. **dark**: High-contrast (presentation)
10. **technical**: Engineering precision (dialogue)
11. **minimal**: Refined, quiet (magazine)
12. **creative**: Bold, expressive (minimal)

## Usage Notes

### Accessibility
- All charts use colour + position encoding
- Patterns available for WCAG compliance (not shown, but accessible via `patterns: true`)
- Semantic HTML with ARIA labels
- Keyboard navigation support

### Performance
- All diagrams server-side rendered (no client-side Mermaid parsing)
- Self-contained HTML (Google Fonts only external dependency)
- File sizes: 65-100KB (highly optimized)
- Fast load times, no flicker

### Customization
- Theme switcher in top-right corner of each example
- All styling via CSS variables (easy to customize)
- Responsive breakpoints at 768px, 1024px, 1440px
- Print-friendly CSS for report layouts

## Generating Your Own

Use the test input documents in `test_inputs/` directory to generate new examples with different LLMs:

1. Read a test document (e.g., `test_inputs/test_1.md`)
2. Call the `analyze_content` MCP tool with the document text
3. Review the analysis and select items to visualize
4. Call `generate_visualization` with selected items and recommended settings
5. Save the HTML output and open in browser

See `scripts/generate_showcase.mjs` for full programmatic examples.

## Technical Details

- **MCP Server**: thought_bubble v0.2.0
- **Rendering**: Server-side SVG (D3.js for charts, beautiful-mermaid for diagrams)
- **Typography**: Google Fonts (theme-specific pairings)
- **Charts**: D3.js v7 with custom renderers
- **Diagrams**: Mermaid via beautiful-mermaid
- **Browser Support**: Modern browsers (Chrome, Firefox, Safari, Edge)
