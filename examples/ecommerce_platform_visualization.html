System Overview

This e-commerce platform is built on a microservices architecture with four core services handling authentication, product management, order processing, and payment integration.

User Service

Handles authentication and user profile management with secure JWT-based authentication

Product Catalog

Manages product inventory, pricing, and stock levels for the entire platform

Order Service

Orchestrates shopping cart functionality and checkout process coordination

Payment Gateway

Integrates with Stripe for secure payment processing and transaction management

Database Schema

Three core entity tables with relationships supporting the platform's data model.

erDiagram Users ||--o{ Orders : places Products ||--o{ Orders : "included in" Users { int id PK string email string password_hash datetime created_at } Products { int id PK string name decimal price int stock } Orders { int id PK int user_id FK decimal total string status }

Users Table

  • id (Primary Key)
  • email (Unique, Indexed)
  • password_hash (Encrypted)
  • created_at (Timestamp)

Products Table

  • id (Primary Key)
  • name (Product Title)
  • price (Decimal)
  • stock (Integer, Inventory Count)

Orders Table
