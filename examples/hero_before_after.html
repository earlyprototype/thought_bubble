# Payment Gateway API

## Overview

The Payment Gateway API provides secure payment processing capabilities for e-commerce platforms.

## Authentication

All API requests require authentication using JWT tokens: - Obtain token from /auth/login endpoint - Include in Authorization header - Tokens expire after 24 hours

## Endpoints

### Process Payment

POST /api/v1/payments/process Request body: { "amount": 99.99, "currency": "USD", "card_token": "tok_123456" } Response: { "transaction_id": "txn_789", "status": "success" }

### Get Transaction

GET /api/v1/transactions/{id} Returns transaction details including status, amount, and timestamp.

## Error Codes

- 400: Invalid request parameters - 401: Authentication failed - 402: Payment required - 500: Server error

## Rate Limits

- 1000 requests per hour per API key - Burst limit: 50 requests per minute
